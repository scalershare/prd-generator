# 必须确认的决策项清单

以下每项不确认都会导致开发返工。在PRD生成前逐项与用户确认。

## 一、用户与权限（8项）

- [ ] 角色定义：有哪些角色？权限边界？
- [ ] 超级管理员：是否需要？能做什么？
- [ ] 数据隔离：多组织/多租户是否需要隔离？
- [ ] 密码方案：默认密码规则？首次强制改密？密码找回？
- [ ] 登录锁定：错误几次锁定？多久？怎么解锁？
- [ ] Token方案：JWT有效期？存哪里？多设备登录？
- [ ] 密码重置：谁能重置谁？重置后默认密码？
- [ ] 账号创建：各级账号怎么创建？手动/批量/自注册？

## 二、数据模型（7项）

- [ ] 核心实体：主要表和关系？
- [ ] 字段完整性：有没有JSON嵌套字段？
- [ ] 唯一约束：防重复的字段组合？
- [ ] 软删除vs硬删除？
- [ ] 历史记录：需不需要保留旧版本？
- [ ] 时区：数据库存UTC还是本地时区？
- [ ] 数据量预估：每张核心表未来可能有多少行？（影响索引策略和是否需要分页/归档）

## 三、业务逻辑（8项）

- [ ] 数据来源：Excel/手动/API/爬虫？
- [ ] 数据导出：需要吗？什么格式？谁能导出？
- [ ] 通知方式：站内消息/邮件/短信/钉钉/飞书？
- [ ] 批量操作：有没有批量处理场景？
- [ ] 异步任务：哪些操作耗时>3秒？等待时展示什么？
- [ ] 操作日志：需要记录谁做了什么吗？
- [ ] 数据备份：多久备份？保留多久？
- [ ] 统计报表：需要哪些统计？给谁看？

## 四、前端交互（6项）

- [ ] 设备适配：移动端+PC？哪个优先？
- [ ] 浏览器兼容：微信内置？IE？
- [ ] 列表页规范：排序/分页/搜索/筛选？
- [ ] 表单验证：必填+格式校验？
- [ ] 错误处理：失败/网络断开怎么展示？
- [ ] 管理后台是否需要移动适配？

## 五、安全（5项）

- [ ] HTTPS：是否必须？
- [ ] 敏感数据：手机号/身份证/支付信息怎么保护？（脱敏展示？加密存储？）
- [ ] 文件上传：允许什么类型？大小限制？
- [ ] API文档：生产环境是否关闭？
- [ ] CORS：允许哪些域名？

## 六、部署与运维（5项）

- [ ] 域名：有没有？需要备案吗？
- [ ] SSL证书：免费还是付费？自动续期方案？
- [ ] CI/CD：自动部署还是手动？
- [ ] 监控告警：通知谁？用什么渠道？
- [ ] 环境划分：开发和生产分开？

## 七、AI相关（如涉及，5项）

- [ ] AI模型：用哪个？
- [ ] Prompt管理：硬编码还是可配置？
- [ ] API成本：月费用预估？
- [ ] 降级方案：AI不可用时怎么办？
- [ ] 并发限制：API限额？需要队列？

## 八、品牌与初始化（4项）

- [ ] 系统名称
- [ ] 品牌配色（无偏好则用默认）
- [ ] 超管初始账号
- [ ] 种子数据

## 九、生产环境专项（8项）

- [ ] 峰值并发：最多多少人同时在线？**（必须用峰值估算法算，不接受拍脑袋）**
- [ ] 性能瓶颈：哪些操作最不能卡顿？
- [ ] 数据丢失后果：能接受丢多长时间的数据？
- [ ] 宕机容忍度：挂1小时影响多大？
- [ ] 技术选型确认：向用户解释关键技术的选择理由，确保理解认可
- [ ] 压力测试：上线前怎么验证扛得住？
- [ ] 运维责任：谁负责日常运维？出问题谁响应？
- [ ] 用户反馈渠道：用户遇到问题怎么反馈？

## 十、测试策略（3项）

- [ ] 后端测试：哪些API需要测试？覆盖正常/异常/权限场景？
- [ ] 前端验收：关键页面的验收标准？（如登录页能正常登录、列表能加载数据）
- [ ] 压力测试场景：模拟什么操作、多少并发、持续多久、通过标准是什么？

## 十一、第三方服务风险（v4.0新增，6项）

- [ ] **第三方服务清单**：列出所有外部依赖（支付/认证/短信/AI API/云存储/CDN/地图/邮件等），每个标注：提供商、计费模型、当前配额上限
- [ ] **冷启动风险**：支付商户号/短信账号/API账号是新开的吗？新账号通常有交易额度限制/频率限制/风控阈值，是否需要提前"养号"预热？
- [ ] **配额与扩容**：各服务的QPS/TPM/日调用量上限是多少？超额时是降级还是直接拒绝？扩容是自动的还是需要人工申请？申请需要多长时间？
- [ ] **SLA对齐**：各第三方服务承诺的可用性是多少（99.9%? 99.5%?）？系统总体SLA不高于最弱环节
- [ ] **费用预警机制**：第三方API调用费用是否有预算上限？超预算时是停止调用还是告警通知？
- [ ] **替代方案**：关键第三方服务（特别是支付和认证）是否有备选提供商？切换需要多长时间？

## 十二、容灾与可用性（v4.0新增，5项）

- [ ] **RTO（恢复时间目标）**：系统宕机后，多长时间内必须恢复？（影响是否需要热备/自动重启/负载均衡）
- [ ] **RPO（恢复点目标）**：最多能接受丢失多长时间的数据？（影响备份频率：每天？每小时？实时？）
- [ ] **灾难恢复验证**：备份是否定期恢复验证？（很多人备份了但从没试过恢复，真出事发现备份是坏的）
- [ ] **事故响应**：P0事故（全站挂）谁第一个响应？多久内必须开始处理？升级路径是什么？
- [ ] **事后复盘**：每次事故是否产出复盘文档？模板是什么？

## 十三、多环境管理（v4.0新增，3项）

- [ ] **环境数量**：开发(Dev)/预发布(Staging)/生产(Prod)三环境？还是更简单的两环境？
- [ ] **环境差异清单**：哪些配置在不同环境不一样？（数据库地址、API Key、日志级别、调试开关等）
- [ ] **Staging环境部署**：是否和生产环境完全一致的配置？用测试数据还是生产数据脱敏副本？

## 十四、数据合规与隐私（v4.0新增，4项）

- [ ] **数据保留政策**：用户注销/删除账号后，数据保留多久？哪些数据必须删除？哪些可以匿名化保留？
- [ ] **用户数据面向学生**：是否涉及未成年人数据？是否需要符合未成年人保护相关要求？
- [ ] **数据导出权**：用户是否有权要求导出自己的所有数据？系统是否需要支持此功能？
- [ ] **隐私弹窗/协议**：面向公众的系统是否需要用户协议和隐私政策？

---

**使用方式**：根据项目类型筛选适用项 → 每项给推荐方案（不是选项） → 用户逐项确认 → 全部确认后才生成PRD。
