# 安全设计标准模板（v4.0）

PRD安全章节应覆盖以下所有层次。根据项目实际情况和技术栈调整具体措施。

## 五层防护（v4.0 扩展为五层）

### 第一层：网络安全

| 措施 | 适用场景 | 说明 |
|------|---------|------|
| HTTPS全站强制 | 所有生产环境 | HTTP→HTTPS 301跳转 |
| 安全头部 | 所有生产环境 | HSTS/X-Content-Type-Options/X-Frame-Options/X-XSS-Protection |
| 隐藏服务器信息 | 所有生产环境 | Nginx: server_tokens off |
| 防火墙 | 所有生产环境 | 仅开放80/443/SSH，数据库端口不对外 |
| IP限流 | 用户量>100 | 反向代理层限制请求频率 |
| WAF | 面向公众的高安全系统 | 云WAF服务 |
| DNS安全 | 所有生产环境 | 上线前降低TTL便于快速切换；稳定后提高TTL减少解析延迟 |

### 第二层：应用安全

| 措施 | 适用场景 | 说明 |
|------|---------|------|
| Token认证(JWT/Session) | 需要登录的系统 | 密钥存环境变量 |
| RBAC权限 | 多角色系统 | 每个API检查角色+数据范围 |
| 租户隔离 | 多租户系统 | 查询强制附加tenant_id |
| SQL注入防护 | 所有系统 | ORM参数化查询 |
| XSS防护 | 有用户输入的系统 | 前端框架转义+后端过滤 |
| CSRF防护 | 使用Cookie认证时 | JWT+Header方式天然免疫 |
| CORS白名单 | 前后端分离 | 仅允许指定域名 |
| 参数校验 | 所有系统 | 强类型验证 |
| 文件上传校验 | 有上传功能 | 限类型+大小+内容（校验文件头，不只看扩展名） |
| API文档关闭 | 生产环境 | 禁用Swagger/docs |
| 登录锁定 | 所有登录系统 | N次失败锁定M分钟 |
| **幂等性设计** | 所有关键写入接口 | 防止网络抖动导致重复提交。使用幂等键(idempotency_key)或数据库唯一约束 |
| **敏感操作二次确认** | 高危操作 | 删除用户/重置密码/批量导出/修改权限等操作，要求输入密码或验证码确认 |
| **API防重放** | 面向公众的高安全系统 | timestamp+nonce+签名，防止Token截获后重放。内部工具可降级为仅用Token+短有效期 |

### 第三层：数据安全

| 措施 | 适用场景 | 说明 |
|------|---------|------|
| 密码加密 | 所有有密码的系统 | bcrypt/argon2，不可逆 |
| 密钥管理 | 所有系统 | 环境变量，不入代码库 |
| 数据库强密码 | 所有系统 | 非默认端口 |
| 定期备份 | 所有生产环境 | cron定时+异地存储 |
| 审计日志 | 多用户系统 | 记录关键操作 |
| 敏感数据脱敏（展示层） | 有个人信息的系统 | 手机号中间*号等 |
| **敏感数据加密存储** | 有身份证/银行卡/敏感PII的系统 | 使用AES-256对敏感字段加密后存储，查询时解密。脱敏只解决"看"的问题，加密解决"存"的问题。注意：加密字段无法直接用SQL查询，需设计查询索引策略（如存储手机号后4位明文用于模糊查找） |
| **备份加密** | 有敏感数据的系统 | 数据库备份文件也要加密存储，防止备份泄露 |

### 第四层：运维安全

| 措施 | 适用场景 | 说明 |
|------|---------|------|
| SSH密钥登录 | 所有服务器 | 禁用密码登录 |
| SSH非默认端口 | 所有服务器 | 改为非22端口 |
| 暴力破解防护 | 所有服务器 | fail2ban等 |
| 容器非root运行 | Docker部署 | USER指令 |
| 依赖漏洞扫描 | 所有系统 | pip-audit/npm audit等 |
| 错误信息脱敏 | 生产环境 | 不暴露堆栈 |
| **Docker镜像安全** | Docker部署 | 使用官方基础镜像，固定版本号（如python:3.11-slim而非python:latest）。生产环境考虑固定SHA256摘要 |
| **密码管理器** | 团队协作 | 服务器密码/API Key等不要存在微信/钉钉聊天记录里，使用密码管理器（1Password/Bitwarden） |

### 第五层：证书与密钥生命周期管理（v4.0新增）

| 措施 | 适用场景 | 说明 |
|------|---------|------|
| **SSL证书到期监控** | 所有HTTPS系统 | Let's Encrypt 90天过期，配置自动续期+续期失败告警。建议在到期前30天、7天、1天各发一次告警 |
| **JWT密钥轮换** | 使用JWT的系统 | 密钥泄露时需在不踢掉在线用户的情况下换密钥：支持同时验证新旧两个密钥，旧密钥设过渡期（如24小时） |
| **API Key轮换** | 使用第三方API的系统 | 泄露时快速替换：设计支持同时生效两个Key做平滑切换（修改.env+重启即可，无需改代码） |
| **数据库密码轮换** | 所有系统 | 定期更换数据库密码时，服务不中断方案：修改密码→更新.env→重启服务（确保连接池配置支持重连） |
| **安全补丁更新** | 所有系统 | 依赖库爆出高危漏洞时，24小时内评估影响，72小时内完成更新。指定专人定期（每月）运行pip-audit/npm audit |

## API错误码标准

| 状态码 | 含义 | 前端处理 |
|-------|------|---------|
| 200 | 成功 | 正常处理 |
| 400 | 参数错误 | 显示字段错误 |
| 401 | 未认证/Token过期 | 跳登录页 |
| 403 | 无权限 | 提示无权限 |
| 404 | 不存在 | 提示未找到 |
| 409 | 数据冲突（乐观锁冲突/幂等重复） | 显示冲突信息，提示刷新后重试 |
| 422 | 校验失败 | 显示错误详情 |
| 429 | 请求过频 | 提示稍后重试，前端做节流(throttle) |
| 500 | 服务器错误 | 通用错误+联系管理员（不暴露堆栈） |
| 503 | 服务不可用/熔断中 | 提示暂时不可用，建议稍后重试 |

## 日志设计规范（v4.0新增）

### 结构化日志格式

所有日志必须使用JSON格式，包含以下字段：

```json
{
  "timestamp": "2025-01-15T10:30:00.123Z",
  "level": "INFO|WARN|ERROR",
  "trace_id": "req-abc123-def456",
  "user_id": "user_001",
  "action": "user.login",
  "duration_ms": 152,
  "status": "success|failure",
  "ip": "192.168.1.1",
  "error_detail": null,
  "extra": {}
}
```

### 链路追踪（Trace ID）

- 每个HTTP请求入口生成唯一trace_id（UUID）
- trace_id在请求全链路传递：API→Service→数据库查询→外部API调用
- 响应头中返回trace_id（X-Request-ID），方便用户报错时提供追踪码

### 日志分级标准

| 级别 | 使用场景 | 示例 |
|------|---------|------|
| **DEBUG** | 开发调试信息，生产环境关闭 | 变量值、SQL语句 |
| **INFO** | 正常业务操作记录 | 用户登录、数据导入完成、AI生成完成 |
| **WARN** | 异常但系统仍可运行 | 外部API重试、缓存未命中、接近配额上限 |
| **ERROR** | 影响功能的错误 | API调用失败、数据库写入失败、支付回调异常 |
| **CRITICAL** | 系统级故障 | 数据库连接断开、磁盘空间不足、安全事件 |

### 日志安全

- **禁止在日志中输出**：密码、Token、身份证号、银行卡号、API Key
- **脱敏规则**：手机号→138****5678，邮箱→z***@example.com
- **日志保留策略**：INFO保留30天，ERROR保留90天，CRITICAL保留180天
- **日志轮转**：单文件不超过100MB，自动轮转，防止磁盘打满

### 必须记录日志的操作

| 操作类型 | 日志级别 | 必须记录的字段 |
|---------|---------|--------------|
| 用户登录/登出 | INFO | user_id, ip, 结果 |
| 登录失败 | WARN | 尝试的用户名, ip, 失败原因 |
| 权限被拒绝 | WARN | user_id, 请求的资源, 当前角色 |
| 数据创建/修改/删除 | INFO | user_id, 操作对象, 变更内容 |
| 外部API调用 | INFO | 目标服务, 响应时间, 状态码 |
| 外部API失败 | ERROR | 目标服务, 错误详情, 重试次数 |
| 支付相关操作 | INFO | 订单号, 金额, 状态变更 |
| 定时任务执行 | INFO | 任务名, 开始/结束时间, 结果 |
| 熔断器状态变更 | WARN | 目标服务, 状态变更(CLOSED→OPEN等) |
